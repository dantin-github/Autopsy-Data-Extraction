Central Gateway Module 
Built on Node.js/Express, the module exposes a unified REST interface to other modules. It acts as the trust hub for the system, responsible for the unified login entry point, role identification, authorisation token generation, email delivery, upload-request reception, and query-request reception. By placing the investigator’s write request and the judge’s verification request into one authorisation and routing framework, it ensures that each role faces different access paths and function boundaries when entering the system. This design prevents coupling identity verification, permission judgment, and request control into Autopsy or a blockchain platform.  

3.5.5 Token-Based Authentication Framework
As the entry point of the system, an appropriate security mechanism for access control and authentication is necessary. The design principle in this study comes from the simulation of real judicial systems, where the investigators cannot conduct forensic activities concerning a case without a valid authorisation credential issued by a supervising authority. The token design in this system technically simulates the authentication process. Furthermore, a role-based authorisation mechanism is also realised with this design. Tokens assigned to investigators and judges decide the write or read-only access for each role, corresponding to the investigative and oversight authority of different legal institutions.
Two authentication paths are configured for investigators and judges/auditors in this system. For the former, a time-limited random OTP token is generated and sent after the initial portal login. Every subsequent transaction from the Autopusy plugin will carry this token to authenticate itself at the API gateway. Once the token expires, it cannot be reused. For the latter, a similar token logic with read-only authorisation is implemented, and no write interface will be exposed to this type of user.
The security design of this research considers basic security requirements as the goal, trying to balance cryptographic complexity and platform performance. Advanced mechanisms, such as asymmetric encryption, can be introduced in production deployment if needed. Nevertheless, the additional security improvements will not substantially influence the research questions in this study.
