# S6.3 — Autopsy upload fault injection (real machine)

Formal install path: **core JAR patch** (`build-patch-core.bat` → `install-patch-core.bat`).  
Save screenshots under `docs/evidence/autopsy-upload/screens/` (suggested names below).

**Common checks (each scenario)**

- `Reports/.../CaseDataExtract/case_data_extract.json` — file exists; `aggregateHash` / body before upload patch intact; **`uploadStatus` is `failed`** (or `cancelled` only for cancel tests, not these three).
- `upload_receipt.json` — **`uploadStatus": "failed"`**; **`errorKind`** matches expectation; **`httpStatus`** if present.
- **Case Data Extract Status** → **Upload Status** — **`Status` = failed**; **Error kind / Error message / HTTP status** rows visible; scenario (a) also shows **red banner** (gateway unreachable).
- **Operations Log** — `UPLOAD_FAILED` (or equivalent) with case id / error kind in detail.
- Gateway logs — optional screenshot for `POST /api/upload` or connection error.

---

## (a) Gateway stopped — `GATEWAY_UNREACHABLE`

1. Start from a **healthy** environment (gateway up, chain as usual).
2. **Stop** the api-gateway process (or block `localhost:3000`).
3. In Autopsy: **Generate Report** → Case Data Extract → enable **Upload**, valid URL, **new OTP**, run.
4. Expect: upload fails; operator message indicates gateway unreachable.

**Expect — `upload_receipt.json`**

- `uploadStatus`: `failed`
- `errorKind`: `GATEWAY_UNREACHABLE`
- `httpStatus`: often `0` (no HTTP response)

**Expect — Monitor**

- Red banner: *Gateway unreachable — check Settings: Gateway Base URL.*
- Error rows populated.

**Screenshot:** `screens/s6.3a-gateway-down-monitor.png` (+ optional receipt snippet).

**After test:** Restart gateway before (b) or (c).

---

## (b) Reuse OTP — `TOKEN_CONSUMED`

1. Gateway **running**; obtain **one** OTP.
2. Run report with **Upload** and that OTP → **first run must succeed** (or at least consume the token).
3. **Without requesting a new OTP**, run report again with **Upload** and the **same** OTP.
4. Expect: second run fails with token already used.

**Expect — `upload_receipt.json`**

- `uploadStatus`: `failed`
- `errorKind`: `TOKEN_CONSUMED`
- `httpStatus`: typically `401`

**Screenshot:** `screens/s6.3b-token-consumed-monitor.png`

---

## (c) Wrong signing password (Contract mode only)

**Skip** if your gateway is **CRUD-only** and you never use Contract upload.

1. Gateway + chain configured for **Contract** upload; police keystore/password known to gateway.
2. In report settings: **Contract** mode, **Upload** on, valid OTP, **intentionally wrong** signing password.
3. Run report.

**Expect — receipt / Monitor**

- `uploadStatus`: `failed`
- `errorKind`: often **`FORBIDDEN`** (HTTP **403**) or **`BAD_REQUEST`** (400) depending on gateway body — either is acceptable if message indicates signing / keystore / password problem.
- Capture gateway JSON `error` / `revertReason` in screenshot or log if present.

**Screenshot:** `screens/s6.3c-wrong-signing-password-monitor.png`
